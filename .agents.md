## 4. API — Fastify Backend (`apps/api`)

### Entry point
`apps/api/src/server.ts` — connects MongoDB, registers routes, serves static web build.

### Auth
All `/api/*` routes require `Authorization: Basic <base64(email:password)>`.  
Exception: `/api/telegram/webhook` (verified by Telegram server origin).

### Routes

#### `apps/api/src/routes/chat.ts`
| Method | Path | Description |
|---|---|---|
| `GET` | `/api/memory` | Returns current dynamic memory string |
| `GET` | `/api/settings` | Returns preferred LLM model |
| `POST` | `/api/settings` | Updates preferred LLM model |
| `POST` | `/api/chat` | Main chat endpoint — runs full pipeline |
| `GET` | `/api/entries/:collection` | Paginated list (`?page=1&limit=50`) |
| `PATCH` | `/api/entries/:collection/:id` | Update a document by MongoDB `_id` |
| `DELETE` | `/api/entries/:collection/:id` | Delete a document by MongoDB `_id` |

**Valid collection names:** `journal` | `gym` | `finance` | `habit`

#### `apps/api/src/routes/telegram.ts`
- Webhook receiver for Telegram Bot.
- Injects messages into `/api/chat` via `fastify.inject()` reusing the full pipeline.
- Supports `/model <name>` slash command to switch LLM at runtime.

### Services

| File | Role |
|---|---|
| `services/db.ts` | MongoDB client, collection getters, `getDistinctExercises()` |
| `services/llm.ts` | LLM router with priority queue + fallback chain |
| `services/memory.ts` | Dynamic memory, structured memory, history, summarization |
| `services/n8n.ts` | Triggers n8n automations via webhook |

### LLM Router (`services/llm.ts`)
- Priority: `targetModel` → `LLM_MODEL` env → fallbacks (`llama3-70b` → `gemma-7b` → `kimi-k2.5`)
- All models accessed via NVIDIA NIM OpenAI-compatible endpoint.

## 5. MongoDB Collections

| Collection | Schema |
|---|---|
| `memory` | `{ userId, dynamicMemory, preferredModel, lastUpdated }` |
| `history` | `{ userId, role, content, timestamp }` |
| `structured_memory` | `{ userId, memory: { preferences, activities, facts }, lastUpdated }` |
| `journal` | `{ entry: string, timestamp }` |
| `gym` | `{ exercise: string, weight: number, unit: "kg"\|"lbs", reps: number, sets: number, type: "standard"\|"top", timestamp }` |
| `finance` | `{ amount: number, method: "cash"\|"card"\|"transfer"\|"crypto"\|"other", category: "fixed"\|"variable"\|"income"\|"savings"\|"investment"\|"other", concept: string, timestamp }` |
| `habit` | `{ name: string, duration: string, comment: string, timestamp }` |

> All entries are inserted with `{ ...data, timestamp: new Date() }`.

## 6. Chat Pipeline (POST `/api/chat`)

```
1. Fetch: memory + structured_memory + distinct exercises (parallel)
2. Fetch recent history (last 15 messages)
3. Build system prompt (soul + tools + known exercises + memory)
4. Save user message to history
5. Call LLM → get responseText
6. Parse <query> tag     → run MongoDB query → re-call LLM with results
7. Parse <db_action> tag → insert document into correct collection
8. Parse <n8n> tag       → fire n8n webhook
9. Save clean assistant response to history
10. Async: checkAndSummarizeMemory() (every 10 messages)
```

## 7. LLM Structured Tags (for Kairo/agents to emit)

All tags must be on their own line. **Never wrap in markdown code fences.**

### Log an entry
```
<db_action>
{ "type": "journal|gym|finance|habit", "data": { ...fields } }
</db_action>
```

### Query past entries
```
<query>
{ "collection": "gym|finance|habit|journal", "query": { "exercise": "curl de bicep", "LIMIT": 5 } }
</query>
```

### Trigger n8n automation
```
<n8n>
{ "action": "ACTION_NAME", "data": { "key": "value" } }
</n8n>
```

### Schedule a reminder
```
<reminder>
{"message": "exact text", "fireAt": "ISO 8601 datetime"}
</reminder>
```

### Google Workspace (Gmail / Calendar)
```
<gmail_list query="..." max=... />
<gmail_read id="..." />
<gmail_draft to="..." subject="..." body="..." />
<calendar_list from="..." to="..." />
<calendar_create summary="..." from="..." to="..." description="..." />
```

> Always include a brief conversational message alongside any tag block.

## 8. Exercise Normalization Rules

- Before logging a gym entry, compare `exercise` name against the `[KNOWN EXERCISES]` list injected into the system prompt.
- The list is fetched live from MongoDB via `getDistinctExercises()` on every chat request.
- If the user's term is a synonym or variant of an existing exercise, use the **existing normalized name**.
- If the name is genuinely new or ambiguous, **ask the user** before creating a new entry: *"Encontré ['curl de bicep'] en tu historial. ¿Lo normalizo con ese nombre?"*

## 9. Frontend — Vite + React 19 (`apps/web`)

### TypeScript rules (strict — enforced by tsconfig.app.json)
- `verbatimModuleSyntax: true` → type-only imports **must** use `import type` or `import { type X }`
- `noUnusedLocals: true` / `noUnusedParameters: true` → no dead variables
- `jsx: react-jsx` → **do NOT import React** as a value just to use JSX; it is injected automatically
- CSS custom properties in `style={}` must be cast: `{ '--var': val } as unknown as CSSProperties`

### API Client (`apps/web/src/api/client.ts`)
- All requests include `Authorization: Basic <token>` header
- Functions: `sendChat`, `getMemory`, `getSettings`, `updateSettings`, `getEntries`, `updateEntry`, `deleteEntry`

### Data Panel (`apps/web/src/DataPanel.tsx`)
- Right sidebar with 4 square icon buttons: 📓 Journal, 🏋️ Gym, 💰 Finance, 🔁 Habits
- Opens `ListPopup` → paginated list (50/page), ✏️ edit, 🗑️ delete
- Edit opens `EditPopup` with schema-aware inputs:
  - `gym.unit` / `gym.type` → `<select>`
  - `finance.method` / `finance.category` → `<select>`
  - `weight`, `amount`, `reps`, `sets` → `type="number"`
  - `journal.entry`, `habit.comment` → `<textarea>`

## 11. Key Patterns for Antigravity

- **Adding a new collection**: add getter in `db.ts` → add to `COLLECTIONS` map in `chat.ts` → add `CollectionMeta` in `DataPanel.tsx` with correct field types → update this file.
- **Adding a new LLM tag**: add regex parser in `chat.ts` pipeline (steps 6–8) → document in section 7 above → update `STRUCTURED_TOOLS_PROMPT` in `packages/core/src/prompts.ts`.
- **Prompt changes** always go in `packages/core/src/prompts.ts`, never hardcoded in routes.
- **Never add secrets** to any file other than `.env.local`.
- **Static top-level imports** are preferred over dynamic `import()` unless circular dependency forces it.
